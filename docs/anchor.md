# Snake 项目 Anchor 文档

## 1. 技术路径（稳定约束）
- 单文件 Web 架构：`index.html` 内聚 `HTML + CSS + JS`，避免拆分导致部署复杂度上升。
- 核心模块保持分层：
  - 路由与 UI：`goScreen`、导航绑定、设置面板。
  - 游戏引擎：`SnakeGame`（移动、碰撞、得分、渲染）。
  - 持久化：`loadStorage` / `saveStorage`。
- 修改原则：优先最小改动修复行为错误，避免无关重构影响稳定性。

## 2. 版本策略
- 采用 `v主.次.修` 语义化版本。
- 类型定义：
  - `feature`：新增功能，通常提升次版本。
  - `bugfix`：缺陷修复，提升修订版本。
  - `refactor`：重构优化，无功能变化，按影响决定次版本或修订版本。

当前版本：`v3.3.1`
